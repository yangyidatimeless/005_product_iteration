## 📋 回应允灿的商业化问题

### 1. 配额累积规则
**决策：不累积，每日重置**

理由：
- **简化逻辑**：不累积的话，数据库只需要记录 `used_today` 和 `last_reset_date`，每天首次使用时检查日期是否变更即可
- **促进活跃**：用户每天登录使用，有利于培养使用习惯
- **行业惯例**：大多数 SaaS 产品（如 Canva、Notion AI）的免费配额都是每日/每月重置，不累积

实现建议：
```python
def reset_quota_if_new_day(user_id: str):
    user = get_user(user_id)
    today = datetime.now().date()
    if user.last_reset_date != today:
        user.used_today = 0
        user.last_reset_date = today
        save_user(user)
```

### 2. 专业版定价策略
**决策：同时支持按月和按年订阅**

| 方案 | 价格 | 相当于每月 | 推荐策略 |
|------|------|------------|----------|
| 按月订阅 | 29 元/月 | 29 元 | 默认选项 |
| 按年订阅 | 299 元/年 | 24.9 元 | 省 17%，标注"推荐" |

理由：
- **灵活性**：不同用户偏好不同，有的喜欢灵活（按月），有的喜欢优惠（按年）
- **现金流**：年付可以提前回笼资金，有利于早期运营
- **转化激励**：年付打折可以激励用户选择长期订阅，降低流失率

数据库设计建议：
```python
class Subscription:
    user_id: str
    plan_type: str  # "monthly" | "yearly"
    status: str  # "active" | "cancelled" | "expired"
    current_period_start: datetime
    current_period_end: datetime
    cancel_at_period_end: bool  # 是否取消自动续费
```

## 📡 实时状态推送方案

**决策：SSE (Server-Sent Events) 为主，WebSocket 为备选**

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **SSE** | 简单、原生支持、自动重连、单向推送足够 | 仅支持单向（服务端→客户端） | 任务状态推送、日志流 |
| **WebSocket** | 双向通信、低延迟 | 实现复杂、需要处理心跳 | 实时聊天、协同编辑 |
| **轮询** | 实现最简单 | 浪费资源、延迟高 | 不推荐 |

**我的建议**：
- 任务状态推送（录制中→转录中→切片中→上传中）用 **SSE**，因为只需要服务端单向推送
- 如果后续需要做实时聊天、协同编辑等功能，再考虑 WebSocket

**前端实现示例**：
```typescript
// composables/useTaskStream.ts
export function useTaskStream(taskId: string) {
  const taskStatus = ref<TaskStatus>('pending')
  const logs = ref<LogEntry[]>([])
  
  const connect = () => {
    const eventSource = new EventSource(`/api/tasks/${taskId}/stream`)
    
    eventSource.onmessage = (event) => {
      const data = JSON.parse(event.data)
      if (data.type === 'status') {
        taskStatus.value = data.status
      } else if (data.type === 'log') {
        logs.value.push(data.entry)
      }
    }
    
    eventSource.onerror = () => {
      // SSE 会自动重连，这里只需处理错误提示
      console.error('SSE connection error')
    }
    
    return () => eventSource.close()
  }
  
  return { taskStatus, logs, connect }
}
```
